# review-article スキル

読者が「欲しい」と感じるアフィリエイト向けHTMLレビュー記事を生成するスキル。

## トリガー
ユーザーが商品レビュー記事の作成を依頼したとき。

---

## 実行ステップ

### Step 1: 情報収集
依頼内容から以下を抽出・整理する。足りない情報はデフォルト値で補う。

- **商品名**（必須）
- **アフィリエイトURL**（なければ `#affiliate-link`）
- **ターゲット層・悩み**（なければ汎用ターゲット）
- **価格・特徴・競合**（任意）
- **SEOキーワード**（なければ「{商品名} レビュー」）

### Step 2: 記事設計
執筆前に心の中で以下を決める（ユーザーに見せない）：

1. **キャッチコピー**: 読者の悩みに刺さる1行
2. **メインベネフィット**: 購入で得られる「変化」を3つ
3. **信頼要素**: 評価・受賞・利用者数など（架空の数字は使わない）
4. **CTA文言**: 行動を促す動詞（「今すぐ試す」「公式で確認する」）

### Step 3: HTML生成
以下の構成で完全なHTMLファイルを生成する。

#### HTML構成（順序厳守）

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <!-- メタ情報・OGP・構造化データ -->
</head>
<body>
  <!-- 1. ヒーローセクション（キャッチ + 評価） -->
  <!-- 2. この記事でわかること -->
  <!-- 3. こんな人におすすめ（共感ゾーン） -->
  <!-- 4. 商品概要・スペック -->
  <!-- 5. 実際に使ってみたレビュー（体験談調） -->
  <!-- 6. メリット・デメリット -->
  <!-- 7. 他商品との比較表（競合情報あれば） -->
  <!-- 8. よくある質問 FAQ -->
  <!-- 9. 総合評価・まとめ -->
  <!-- 10. 購入CTA（ボタン） -->
</body>
</html>
```

### Step 4: 品質チェック
生成後に以下を確認する：

- [ ] CTAボタンが上部・中間・末尾の3箇所以上にある
- [ ] アフィリエイトリンクに `rel="nofollow sponsored"` がある
- [ ] 構造化データ（Product / Review / FAQPage）が含まれている
- [ ] レスポンシブCSS（max-width, padding, font-size調整）がある
- [ ] 誇大表現・薬機法リスクのある断言がない
- [ ] メリット・デメリットの両方を正直に記載している

### Step 5: 保存
`articles/{商品名スラッグ}-review.html` に保存する。

---

## 出力形式

完全なHTML1ファイルを出力する。  
外部CSSフレームワーク不使用（`<style>`タグでインライン記述）。  
日本語記事として、自然な文体で執筆する。

---

## ライティングのポイント

### 冒頭（読者を引き込む）
```
「{悩み}で悩んでいませんか？」
→ 共感 → 「この記事では{商品名}を実際に試した結果をお伝えします」
```

### 体験談パート（信頼を作る）
- 使用前の状態 → 使用中の変化 → 使用後の結果 の3段構成
- 具体的な日数・数値を入れる（「3日目から変化を感じた」）
- ネガティブな点も1つ正直に入れると信頼度UP

### CTA（クリックさせる）
```html
<!-- 良いCTA例 -->
<a href="{URL}" rel="nofollow sponsored noopener" target="_blank" class="cta-button">
  公式サイトで詳細を確認する →
</a>
```
- 「買う」より「確認する」「試す」「詳しく見る」の方がハードルが低い

### FAQ（SEOと不安解消）
- 「本当に効果あるの？」「副作用は？」「返金保証は？」など購入障壁となる質問を選ぶ
- FAQPage構造化データと組み合わせる
